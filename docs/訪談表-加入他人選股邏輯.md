# 訪談表：把別人的選股邏輯放進系統

> 用途：訪談一位有自己一套方法的投資人，把他的判斷標準翻譯成系統參數與因子。
> 訪談後可以直接跑三年回測，用數據驗證他的方法。

## 訪談前先講清楚的兩件事

**第一，這不是在質疑他。** 回測是提供他一個他自己可能沒有的東西 ——
過去三年、上百次進場的統計結果。多數人憑印象記得贏的那幾次。

**第二，答案要具體到可以寫成程式。**

| ❌ 不能用 | ✅ 可以用 |
|---|---|
| 「我看基本面好的」 | 「近兩年 EPS 都要 > 8 元」 |
| 「等它站穩再進」 | 「連續 3 天收在月線上」 |
| 「量要出來」 | 「當日量 > 5 日均量 1.5 倍」 |
| 「跌太多就跑」 | 「跌破成本 5% 停損」 |

問到模糊的答案時追問：**「那你怎麼知道『站穩』了？如果我要幫你寫成程式，
要怎麼判斷？」**

---

## 第一部分：直接對應現有參數

這些問完就能立刻做出一個他的專屬策略檔並回測。

### 基本面門檻

| 問題 | 對應參數 | 目前預設 |
|---|---|---|
| EPS 至少要多少才考慮？ | `eps_threshold` | 5 元 |
| ROE 至少要多少？ | `roe_threshold` | 15% |
| 基本面不過的股票，完全不碰嗎？<br>還是技術面夠強也可以？ | `fundamental_pass_required` | 是（完全不碰）|

### 風險控管

| 問題 | 對應參數 | 目前預設 |
|---|---|---|
| 停損設幾 %？ | `stop_loss` | 8% |
| 停利設幾 %？ | `target_return` | 10% |
| 通常抱多久？ | `hold_days` | 20 個交易日 |
| 單一檔最多放多少資金？ | `position_size_pct` 上限 | 20% |

> 停損停利這組最關鍵 —— 它直接決定回測結果。
> 如果他說「看情況」，追問「最常見的情況是幾 %」。

### 三個評分項的比重

問法：**「基本面、技術面、歷史勝率，這三個對你來說重要性怎麼分配？
給我三個加起來 100 的數字。」**

| | 對應參數 | 目前預設 |
|---|---|---|
| 基本面 | `weight_fundamental` | 30 |
| 技術面 | `weight_technical` | 30 |
| 歷史回測勝率 | `weight_backtest` | 40 |

### 技術訊號取捨

問法：**「這四個你會看哪些？不看的我就關掉。」**

| 訊號 | 對應參數 |
|---|---|
| 均線多頭排列（站上月線、季線）| `use_ma_alignment` |
| 布林通道下軌反彈 | `use_bollinger_bounce` |
| KD 黃金交叉 | `use_kd_golden_cross` |
| MACD 翻正 | `use_macd_bullish` |
| 量價型態（倍量、梯量、放量滯漲…）| `use_volume_patterns` |

### 大盤環境

| 問題 | 對應參數 |
|---|---|
| 大盤走弱時你會停手嗎？還是照做個股？ | `market_filter_enabled` |
| 用什麼判斷大盤走弱？（月線？季線？）| `market_filter_ma_period` |

## 第二部分：他有但系統沒有的

問法：**「你看盤時還會看什麼，是上面沒提到的？」**

以下是已知系統沒有、但常被提到的。有的話記下他的**具體判斷條件**：

- [ ] **RSI** — 幾以下算超賣？幾以上算超買？
- [ ] **OBV** — 怎麼用？看背離還是看趨勢？
- [ ] **W 底 / 頭肩底** — 兩個低點要多接近？頸線怎麼抓？
- [ ] **三角收斂** — 觀察幾天？突破要帶多少量？
- [ ] **籌碼** — 外資連買幾天？投信認不認？（系統有這些因子，但預設關閉）
- [ ] **月營收** — 年增率多少算好？連續幾個月？
- [ ] **本益比 / 淨值比** — 有絕對門檻，還是看歷史區間相對位置？
- [ ] **其他**：＿＿＿＿＿＿＿＿

> 系統已內建但目前關閉的：籌碼、成長、營收、評價共 13 個因子。
> 如果他重視這些，開啟就好，不用重寫。

---

## 第三部分：容易被忽略但很重要的

這幾題常常問出最有價值的東西：

**1. 「你什麼情況會不買？」**
排除條件往往比進場條件更能定義一個人的方法。

**2. 「最近一次你覺得系統性判斷錯誤是什麼時候？後來怎麼調整？」**
會問出他踩過的坑，那通常是最珍貴的部分。

**3. 「你怎麼決定買多少？」**
部位管理常常比選股更影響最終績效。

**4. 「你會分批進場嗎？」**
系統目前假設單筆進場，若他習慣分批，回測的意義會不一樣。

**5. 「這套方法在什麼行情下會失效？」**
誠實的人會有答案。沒有答案的方法要小心。

## 訪談後的流程

1. **填成策略檔** → `strategies/<他的名字>.json`
2. **跑回測比較**

   ```bash
   uv run python main.py --limit 20        # 預設策略
   # 對照他的策略跑一次，比較勝率與平均報酬
   ```

3. **一起看結果**

   結果可能有三種，每種的處理方式不同：

   | 結果 | 意義 | 怎麼做 |
   |---|---|---|
   | 他的方法勝率明顯較高 | 值得採用 | 考慮切換或混合 |
   | 差不多 | 兩者本質相近 | 用他熟悉的，執行才會確實 |
   | 他的方法較差 | **先別下結論** | 見下方 |

### 如果回測結果比較差

**不要直接說「你的方法沒用」。** 幾個更可能的原因：

- **回測期間的行情不利於他的風格**（例如他擅長空頭，但這三年是多頭）
- **有些判斷沒被翻譯出來** —— 他實際會看的東西沒完全講出來，或講不出來
- **樣本數太少** 沒有統計意義
- **他的優勢在執行紀律而非選股規則** —— 這是回測測不出來的

比較有建設性的問法：**「回測顯示這三年這樣做勝率是 X%，跟你自己的感覺
差多少？如果差很多，是不是有什麼判斷我們沒寫進去？」**

## 一個誠實的提醒

**能寫成規則的部分，回測能驗證；寫不成規則的部分，我們無從驗證。**

如果他的方法有很大一部分是「盤感」，那用 LLM 模仿他的口吻是可以的，
但那只是**像他在講話**，不代表判斷品質跟他一樣，而且**無法回測**。

建議的分工：
- **能規則化的** → 寫成參數與因子，可驗證
- **純風格的** → 最多放進 AI 的說話方式，不要讓它影響評分

不要讓沒被驗證過的東西影響實際的買賣判定。
