# 用 AI 把開源專案變成能用的系統

> 一天內把一個 GitHub 專案，變成會自動推播、能對話、有網頁儀表板的完整系統
>
> 簡報素材｜每個 `##` 約對應一張投影片

---

## 起點：一個開源專案

fork `kevin801221/stock-strategies-only`

**它已經有的**
- 台股訊號引擎（基本面 × 技術面 × 三年回測）
- 每日 Telegram 推播
- GitHub Actions 排程

**但我想要的是**
- 能**對話**的 AI 機器人，不只是單向推播
- 一個**網頁儀表板**
- 盤中也能提醒

---

## 落差分析

實際讀完程式碼後發現三個缺口：

| 想要 | 現況 |
|---|---|
| 對話 | 只有 `send_telegram()`，沒有收訊息的能力 |
| 用我的資料 | 硬綁 Google Sheet，我沒有憑證 |
| AI 判讀 | Gemini 只用在網頁端「生策略」，跟推播無關 |

**關鍵決定**：不從零重寫。訊號引擎品質不錯，只加「對話」和「呈現」兩層。

---

## 建置流程

```
① 環境           uv + Python 3.12
② Telegram 機器人  長輪詢收訊 + Gemini 對話腦
③ Google Sheet    自動化建立服務帳號
④ 觀察名單        機械規則選出 100 檔
⑤ 網頁儀表板      靜態產生 + 登入保護
⑥ 盤中功能        持倉監控 + 訊號預覽
```

每一階段完成就**用真實憑證打一次真實 API** 驗證，不等全部串完才測。

---

## 核心設計：讓 AI 呼叫工具，而不是憑記憶回答

```
你在 Telegram 打字
      ↓
是斜線指令？→ 直接處理（快、不耗 API 額度）
      ↓ 不是
Gemini 對話腦
      ↓ 模型自己決定要呼叫哪個工具
6 個工具 → 訊號引擎 / 大盤 / 觀察名單 / 績效
```

**System prompt 明確禁止模型憑印象回答股市數字。**

LLM 對股價的幻覺特別危險 —— 寧可回「查詢失敗」，也不要編一個看起來很像的假數字。

---

## 踩到的坑（一）：測試全過，實際全掛

6 個工具的 schema 驗證**全數通過**，但真正呼叫時**全部失敗**。

**原因**：`from __future__ import annotations` 讓型別註記在執行期變成字串，
SDK 內部的 `isinstance(value, annotation)` 就爆了。

**為什麼難查**：SDK 把例外包成「工具的回傳值」餵回模型，
模型只好回「查詢失敗」—— 看起來像股票代號有問題。

> **教訓**：schema 驗證過 ≠ 能用。每個外部服務都要用真實呼叫驗證。

---

## 踩到的坑（二）：清單裡有的模型不能用

`gemini-2.5-flash` 出現在 `models.list()` 裡，
但新申請的 API key 呼叫會回 **404 no longer available to new users**。

清單與實際權限不一致，**只能靠實際呼叫才知道**。

而且要測**你真正需要的能力** —— 這個專案需要 function calling，
就不能只測普通對話會不會回應。

---

## 踩到的坑（三）：差點把憑證推上公開 repo

我要求把 `.env` 推上 GitHub 時，AI 拒絕了，並指出：

- repo 是 **PUBLIC**
- `.env` 含 Google 私鑰、Telegram token、API keys
- GitHub 上有機器人 24 小時掃描新 commit，**幾分鐘內就會被撿走**

正確做法是 **GitHub Secrets**（加密儲存、log 自動遮蔽、fork 拿不到）。

> 後來還發現 `.gitignore` 只擋 `.env`，**擋不到 `.env.bak`** —— 備份檔一樣含私鑰。

---

## 安全設計：把檢查做進流程，而不是靠記得

儀表板會部署到公開網頁，所以**每次建置都掃描輸出**，
比對所有環境變數值，命中就中止並刪除輸出。

```
✅ index.html + 100 個詳情頁
✅ 憑證外洩檢查通過
```

不是「相信程式碼不會碰到 secrets」，而是**每次實際驗證一遍**。
未來任何改動意外洩漏，都會讓部署失敗而不是悄悄上線。

---

## 網頁儀表板

**不用前端框架** —— 用 Python 直接產生靜態 HTML

- 零依賴、CI 不必 `npm install`
- K 線圖是純 Python 產生的 SVG
- 100 檔含圖表約 2 MB

**存取控制**：Cloudflare Access（email 驗證碼登入）

> 為什麼不用 GitHub Pages：免費版**一律公開**，即使 repo 是 private。
> 而靜態頁面**做不到**真正的密碼保護 —— 用 JS 判斷密碼是假的，看原始碼就破了。

## 三個帶得走的原則

**1. 每接一個外部服務，就用真實呼叫驗證一次**
單元測試過、schema 驗證過，不代表能用。今天 5 個 bug 有 4 個是整合才炸的。

**2. 憑證永遠不進 git**
需要給雲端用就走該平台的 secrets 機制。不確定就 `git check-ignore -v <檔名>`。

**3. 把檢查做進流程，而不是靠記得**
「我檢查過了」會失效，「每次建置都自動檢查」不會。

---

## 誠實的限制

**這是量化計算工具，不是投資建議。**

- 回測假設嚴格照規則執行、不滑價、不手動干預 —— 實際結果一定比較差
- 沒有計入手續費與證交稅（來回約 0.6%）
- 盤中訊號**沒有回測支撐**，只能當觀察
- 真正的驗證要看實際追蹤 20 個交易日後的成績

**憑證會過期、API 會改版。** 排程失敗時推播只是安靜地不來 ——
沉默是最容易被忽略的失敗模式。
